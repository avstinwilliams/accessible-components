Redesigned System Preferences mockup

In my redesign, I noticed that once a radio button is selected, users cannot deselect it. It's fairly common for someone to accidentally select an option and have no way to undo their choice, so I decided to add a small red "x" to the right of the text of the selected radio button. This maintains the efficiency of System Preferences' current design, while improving usability. In terms of learnability, 'x' typically represents cancellation, and the choice to make it red further enforces that fact. Memorability is not necessary in this case, as the x will be present as long as something is selected.

Part 4: Reflections

  • The radio buttons I observed performed well in usability, but accessibility varied. Google Forms provided strong keyboard navigation where System Preferences lacked it entirely.
  • My design incorporated an undo selection option to let users deselect a radio button, addressing a missing accessibility consideration in real-world applications.
  • Following the social model of disability, my redesign reduces the mismatch between users and the interface by making selection more flexible for all users, including those with motor impairments.
  • For example, System Preferences completely lacks support for keyboard users, meaning those who have poor motor functions and struggle to use a mouse would run into trouble.
  • Similarly, Google Forms’ lack of explicit selection announcements for screen readers makes it harder for visually impaired users to navigate form inputs effectively.
  • Among the different input methods, mouse and touch interactions are commonly prioritized in UI design, which creates barriers for users who rely on assistive technologies.
